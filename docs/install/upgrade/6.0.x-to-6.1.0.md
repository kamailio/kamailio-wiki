# Upgrade Kamailio from v6.0.x to v6.1.0

The page contains the details about the changes that were made to old
components during the development of v6.1.0 compared with what existed
in v6.0.x. It does not include the brand new modules, focusing on how to
upgrade database and configuration file from v6.0.x to run with Kamailio
6.1.0.

## Previous Stable Release

If you look for the guidelines to upgrade to previous stable release,
see:

- [Upgrade Kamailio v5.8.x to v6.0.0](5.8.x-to-6.0.0.md)

## Remarks

Following tokens are used to mark the changes:

- INF - the change doesn't really have any direct impact - no action
    required in old config
- CPM - the change was used with the new value via explicit parameter
    in default config file from old releases
- RCM - the change is recommended to be done if you had an explicit
    different or lower value for this parameter in old config
- UPG - the change has to be done, older config does not work any
    longer

## Database

### MySQL Database Structure

Run following SQL statements in MySQL client to upgrade database
structure from v6.0 to v6.1:

``` sql
ALTER TABLE acc_cdrs CHANGE COLUMN `duration` `duration` FLOAT(13,6) DEFAULT 0 NOT NULL; -- # was `duration` FLOAT(10,3) DEFAULT 0 NOT NULL

UPDATE version SET table_version = 3 WHERE table_name = 'acc_cdrs';
```
